# STAR_CHAIN_AND_MERIDIAN_REVEAL_MOTION_SPEC_V1

## 0. Document Status

- STATUS: FROZEN_FOR_ART_DIRECTION
- OWNER: ART / PRODUCT / TECH SHARED
- PURPOSE: 冻结"信物档案页"中星链、经络、节点点亮、待点亮补播、暗态/亮态/已收录态的视觉与动效规则
- SOURCE CONCEPT:
  - 打卡掉落的是"宝物"
  - "信物"是宝物对应的别名 / 图谱节点
  - 用户在"信物档案页"查看星链 / 经络中的节点点亮结果

---

## 1. Core Visual Principle

信物档案页不是普通图鉴，也不是游戏战力盘。

它应呈现为：

```text
东方图谱
古籍感
天文观测感
经络流动感
克制仪式感
节点收录感
```

禁止方向：

```text
强科幻HUD
霓虹游戏UI
五角星政治符号
亮金爆炸特效
金属徽章贴面
抽奖稀有奖励感
战力升级盘
医学教材风经络图
```

---

## 2. Two Visual Systems

信物档案页包含两套主视觉系统：

```text
A. 星链系统
B. 经络系统
```

二者共享"节点点亮"逻辑，但视觉气质不同。

---

## 3. Star Chain System

### 3.1 Definition

星链系统用于承载星宿 / 星位类信物。

示例：

```text
宝物：莲花印
信物别名：角宿一（室女座 α，Spica）
图谱类型：星链
节点：角宿一
```

### 3.2 Visual Structure

星链图谱由以下元素组成：

```text
星点节点
星点之间的细线
星宿分组
四象区域
暗态星点
已点亮星点
待点亮星点
节点标签
```

### 3.3 Visual Reference

用户给出的参考图中，"角宿"可理解为：

```text
两个星位
一条细线连接
其中一个节点被点亮
```

这类结构可作为星链最小视觉单元：

```text
星点 — 星点
```

要求：

* 星点不要画成五角星
* 星点应为圆点 / 微孔 / 星孔 / 光点
* 连线应细、克制、像观测线或古星图连线
* 点亮节点可出现柔和光晕
* 未点亮节点保持暗淡但可识别

## 4. Star Chain Node States

### 4.1 未获得态

视觉：

* 暗色圆点
* 低透明度
* 细微颗粒质感
* 不发光或仅极弱光
* 连线可见但压低存在感

状态含义：

```text
用户尚未获得该信物
```

---

### 4.2 已获得但未查看态

视觉：

* 节点比未获得态略亮
* 可有微弱呼吸
* 不进入正式常亮
* 可在节点边缘出现很细的待激活环

状态含义：

```text
用户已获得对应宝物，但还未进入档案页看过点亮动画
```

---

### 4.3 点亮中态

视觉：

* 一缕流光沿星链连线移动
* 流光汇入目标星点
* 星点由内向外亮起
* 出现柔和扩散圈
* 节点标签短暂浮现

状态含义：

```text
当前节点正在执行新信物点亮动画
```

---

### 4.4 已点亮态

视觉：

* 稳定常亮
* 光晕克制
* 节点可点击
* 标签可显示或在点击后显示
* 连线可局部增强

状态含义：

```text
该信物已经完成档案收录
```

---

## 5. Star Chain Reveal Animation

### 5.1 单节点点亮流程

动画顺序：

```text
1. 目标节点暗点微闪
2. 上游星链出现微弱流光
3. 流光沿连线向目标节点移动
4. 流光汇入目标节点
5. 节点由暗转亮
6. 节点外圈出现柔和扩散
7. 节点标签浮现
8. 节点进入稳定常亮
```

建议时长：

```text
总时长：1.6s ~ 2.4s
```

气质要求：

```text
像古星图被重新点亮
不是抽卡爆光
不是科技雷达扫描
```

---

### 5.2 多节点顺序点亮

当存在多个未查看信物时：

```text
按 collectedAt 升序
一个节点完成后再播放下一个
```

节奏：

```text
1~5 个：完整播放
6~15 个：前 3~5 个完整播放，后续加速
15 个以上：批量归档点亮模式
```

视觉要求：

* 不要所有节点同时闪爆
* 不要只点亮最后一个
* 多节点之间应有"星图逐渐苏醒"的感觉

## 6. Meridian System

### 6.1 Definition

经络系统用于承载穴位 / 经络类信物。

示例：

```text
宝物：青铜针符
信物别名：百会穴
图谱类型：经络
节点：百会
```

### 6.2 Visual Structure

经络图谱由以下元素组成：

```text
经络路径线
穴位节点
经络分组
暗态穴位
待点亮穴位
已点亮穴位
流动光
节点标签
```

### 6.3 Visual Direction

经络不应画成现代医学教材图。

推荐气质：

```text
古法经脉图
纸本线描
淡墨路径
内气流动
柔和节点
东方身体观
```

禁止：

```text
霓虹医疗UI
红蓝血管图
人体解剖教材风
过度写实器官图
```

---

## 7. Meridian Node States

### 7.1 未获得态

视觉：

* 暗淡小节点
* 经络线细弱
* 节点可见但不强调

---

### 7.2 已获得但未查看态

视觉：

* 节点略亮
* 经络线局部有微弱流动预兆
* 节点周围可有淡淡脉冲

---

### 7.3 点亮中态

视觉：

* 光流沿经络路径移动
* 光流汇入目标穴位
* 穴位节点由暗转亮
* 出现一圈柔和脉冲
* 节点标签浮现

---

### 7.4 已点亮态

视觉：

* 节点稳定发光
* 经络线对应段落略亮
* 节点可点击查看详情

---

## 8. Meridian Reveal Animation

### 8.1 单节点点亮流程

动画顺序：

```text
1. 经络线局部出现微弱流动
2. 光流沿路径推进
3. 光流汇入目标穴位
4. 穴位节点由暗转亮
5. 节点出现柔和脉冲
6. 节点标签浮现
7. 节点进入稳定点亮态
```

建议时长：

```text
总时长：1.8s ~ 2.8s
```

气质要求：

```text
像气脉贯通
不是电流冲击
不是医疗仪器扫描
```

## 9. Shared Archive Reveal UX

### 9.1 新获得单个信物

用户进入档案页后：

```text
自动定位对应图谱
自动聚焦对应节点
播放点亮动画
显示收录完成提示
节点常亮
```

---

### 9.2 多个未查看信物

用户进入档案页后：

```text
读取待点亮队列
按获得时间依次点亮
全部完成后显示汇总
```

汇总文案示例：

```text
本次档案更新完成
新增点亮：3
星链新增：2
经络新增：1
```

---

### 9.3 用户跳过动画

视觉策略：

```text
剩余节点快速点亮
不逐个完整演出
最终展示汇总
```

按钮文案：

```text
跳过动画
一键完成点亮
```

## 10. Color And Material Direction

### 10.1 Base Background

推荐：

```text
米白纸本
浅褐古籍底
暗青灰夜空底
低饱和宣纸纹理
```

避免：

```text
纯黑科幻背景
高饱和蓝紫霓虹
强金属质感
```

---

### 10.2 Star Chain Color

推荐：

```text
暗青
灰蓝
旧金
淡白星光
微暖金光
```

星点亮态：

```text
柔白 + 旧金边缘
```

避免：

```text
亮金爆炸
荧光蓝
纯五角星形状
```

---

### 10.3 Meridian Color

推荐：

```text
淡墨
温金
朱砂暗红
浅赭
柔白节点光
```

避免：

```text
医疗蓝
荧光绿
血管红蓝强对比
```

## 11. Motion Timing Guide

### 11.1 Star Chain

```text
暗点预热：300ms
流光移动：600ms ~ 900ms
节点点亮：400ms
标签浮现：300ms
稳定收束：300ms
```

---

### 11.2 Meridian

```text
经络预热：400ms
光流推进：700ms ~ 1000ms
穴位点亮：400ms
脉冲扩散：400ms
稳定收束：300ms
```

---

### 11.3 Multi Reveal Interval

```text
节点间隔：300ms ~ 600ms
加速模式节点间隔：120ms ~ 240ms
```

## 13. Lottie / Canvas / CSS Guidance

### 13.1 V1 Recommendation

V1 可以先采用：

```text
Canvas / 小程序原生动画 / CSS animation
```

不强制 Lottie。

### 13.2 Lottie Use Case

适合用 Lottie 的部分：

```text
节点点亮微动效
流光脉冲
收录完成小提示
```

不建议把完整图谱做成单一 Lottie，因为后续节点状态需要动态控制。

## 14. Asset Rules

### 14.1 星链资产

需要：

```text
星点节点基础样式
星点亮态样式
连线样式
流光样式
节点标签样式
```

### 14.2 经络资产

需要：

```text
经络线基础样式
经络流光样式
穴位暗态
穴位亮态
穴位脉冲
节点标签
```

### 14.3 禁止

```text
不要把所有节点画死在一张不可交互图片里
不要让前端无法单独控制节点状态
不要让节点无法按队列顺序点亮
```

---

## 15. Minimal V1 Visual Acceptance

V1 最小可验收版本必须支持：

```text
1. 至少一组星链，两颗星位，一条连接线
2. 星位支持未获得 / 待点亮 / 已点亮
3. 至少一条经络线，多个穴位节点
4. 穴位支持未获得 / 待点亮 / 已点亮
5. 新节点可播放点亮动画
6. 多节点可依次点亮
7. 已点亮节点保持常亮
```

## 18. Cross-Team Notes

### For Product

* 不再说"掉落信物"
* 应说"获得宝物，激活信物"
* 信物档案页负责图谱点亮

### For Tech

* 每个节点必须可被独立控制状态
* 动画必须能按 pendingRevealQueue 顺序播放
* 不可把图谱做成不可交互静态大图

### For Art

* 宝物实体视觉与信物图谱视觉要分开
* 星链与经络可共享点亮语言，但不能完全同质化
* 视觉应克制、有东方档案感

---

## 19. Final Freeze

本文件冻结以下视觉方向：

```text
星链 = 东方古星图式节点连线系统
经络 = 古法经脉式节点路径系统
点亮 = 节点被信物收录后的仪式性激活
多节点 = 按队列依次点亮，不同时爆亮
图谱资产 = 必须支持前端独立控制节点状态
```
