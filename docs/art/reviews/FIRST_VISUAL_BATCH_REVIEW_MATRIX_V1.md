## Review Rules

Each score uses 1–5.

* Style Fit
* Cultural Fit
* Product Fit
* Technical Usability
* Motion Potential
* Clarity
* Risk Control

Decision:

* 28+ = PASS candidate
* 21–27 = REVISE
* <21 = REJECT
* REFERENCE_ONLY may be used when an image is useful as inspiration but not accepted as final

## Global Rejection Rules

Reject immediately if any candidate contains:

* five-pointed star
* political symbol
* metallic badge look
* bright gold explosion
* gacha reward style
* sci-fi HUD
* neon game UI
* medical textbook style
* anatomy / organs
* red-blue vein diagram
* non-controllable static diagram unsuitable for frontend
