# AR游伴 商业接入标准 V1

## 一、接入对象

- 景区
- 展馆
- 商家活动
- 城市文化点位

## 二、接入方式

### 1. 空间点位绑定

每个接入方必须提供：
- 地理空间点位
- 或 AR Marker
- 或 场景触发条件

### 2. 信物定义

接入方可定义：

- relic_name（信物名称）
- relic_meaning（意义）
- relic_rarity（稀有度）
- relic_trigger_condition（触发条件）

### 3. 行为触发规则

支持：

- 用户进入区域
- 用户点击入口
- AR识别成功
- 时间触发

### 4. 数据回流

系统输出：

- 用户探索路径
- 信物触发记录
- 空间停留时间
- 行为事件链

## 三、运营能力

支持：

- 活动投放
- 信物发放
- 空间任务配置（非游戏化）
- 用户路径分析

## 四、系统边界（重要）

不做：

- 抽卡
- 等级体系
- 游戏数值

只做：

- 空间
- 行为
- 信物
- 关系

## 五、核心价值

AR游伴 = 把现实空间变成可记录的体验结构
