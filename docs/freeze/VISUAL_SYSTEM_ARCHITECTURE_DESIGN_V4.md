## 3. 输出要求

在每次视觉设计任务中，以下 6 项产出是**强制性的**，缺少任意一项即为不完整设计：

### 3.1 全系统页面地图（必需）

```
FULL SYSTEM PAGE ARCHITECTURE
└── 涵盖所有页面，不包括独立页面设计

要求：
  - 列出系统中所有页面的完整清单
  - 不得遗漏任何页面
  - 不得将任意页面视为"独立设计单元"
```

### 3.2 页面数量定义

```
PAGE COUNT DEFINITION
└── 明确系统中应当有多少个页面

要求：
  - 当前版本的总页面数
  - 每页的页面 ID
  - 页面类型（landing / explore / detail / utility / overlay）
  - 设计状态（designed / in-progress / pending）
```

### 3.3 页面关系图

```
PAGE RELATIONSHIP GRAPH
└── 页面之间的导航、层级和依赖关系

要求：
  - 父/子页面层级
  - 横向导航关系
  - 模态/浮层从属关系
  - 数据依赖关系
  - 状态传导路径
```

### 3.4 用户旅程流

```
USER JOURNEY FLOW
└── 用户如何流经整个系统

要求：
  - 从入口到各终点的完整路径
  - 不同用户类型的路径分歧（访客 / 已登录 / 活跃探索者）
  - 回退路径与死胡同处理
  - 状态转换触发点
```

### 3.5 全局视觉一致性规则

```
GLOBAL VISUAL CONSISTENCY RULES
└── 跨所有页面强制执行的视觉规则

要求：
  - 设计令牌系统（颜色、字体、间距、圆角）
  - 组件复用清单（不得在每个页面重新实现）
  - 跨页面动画连续性
  - 加载 / 空状态 / 错误状态的统一处理
  - 资产使用规则（同一资产不得被不同页面以不同方式使用）
```

### 3.6 每个页面的结构布局定义

```
PAGE-LEVEL STRUCTURAL LAYOUT
└── 系统中每个页面的布局定义

要求：
  - 页面 ID
  - 层结构（骨架 → 主内容 → 叠加层 → 固定层）
  - 关键区块描述
  - 区块间关系（堆叠 / 重叠 / 条件显示）
  - 数据绑定点
  - 资产绑定点
```

## 4. 硬性规则

```
┌─────────────────────────────────────────────────────────────────────┐
│ RULE-001                                                           │
│ No isolated page design allowed                                    │
│ ─────────────────────────────────────────────────────────────────── │
│ 禁止对任意页面进行孤立设计。                                         │
│ 每个页面的视觉设计必须在其与其他所有页面的关系上下文中进行。          │
│ 违反：孤立设计的页面视为无效，必须重做。                              │
├─────────────────────────────────────────────────────────────────────┤
│ RULE-002                                                           │
│ No single-page-first design allowed                                │
│ ─────────────────────────────────────────────────────────────────── │
│ 禁止"先从单个页面开始设计，再考虑系统集成"的方法。                  │
│ 必须先定义 FULL SYSTEM PAGE ARCHITECTURE，再设计具体页面。          │
│ 违反：单页优先的设计方法视为无效。                                   │
├─────────────────────────────────────────────────────────────────────┤
│ RULE-003                                                           │
│ System coherence is mandatory                                      │
│ ─────────────────────────────────────────────────────────────────── │
│ 系统连贯性是强制性的设计约束。                                       │
│ 所有页面必须共享同一视觉语言、组件库和设计令牌。                     │
│ 页面之间不得出现不一致的视觉风格、间距系统或交互模式。               │
│ 违规：不一致的页面视为设计缺陷。                                     │
├─────────────────────────────────────────────────────────────────────┤
│ RULE-004                                                           │
│ User experience continuity is highest priority                     │
│ ─────────────────────────────────────────────────────────────────── │
│ 用户体验连续性高于任何单个页面的视觉独特性。                         │
│ 跨页面导航时，视觉过渡、信息层级、交互模式必须连贯。                 │
│ 如果页面独特性与连续性冲突，连续性优先。                             │
└─────────────────────────────────────────────────────────────────────┘
```
