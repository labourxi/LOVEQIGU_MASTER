# LOVEQIGU / AR游伴
# UI CONTRACT SYSTEM V1（冻结版）

## 一、系统页面总结构

系统包含6个核心页面：

- PAGE_01：Landing（爱企谷首页）
- PAGE_02：探索地图
- PAGE_03：信物 / 宝物系统（Relic）
- PAGE_04：回响系统（Echo）
- PAGE_05：数字藏品（Collectible）
- PAGE_06：个人中心

---

## 二、全局UI结构原则（强制）

所有页面必须遵守三层结构：

### L1：视觉层（世界场景）
- 爱企谷 / 星图 / 叙事空间背景

### L2：系统状态层（数据展示）
- 探索进度
- 信物数量
- 藏品数量
- 权益数量

### L3：行为层（操作入口）
- 登录
- 探索
- 查看
- 触发AR

---

## 三、PAGE_01 Landing（爱企谷首页）

### 目标：
用户进入爱企谷并理解系统状态

### 必须包含：

#### L1 视觉层
- 爱企谷主世界视觉
- 门 / 光 / 路径隐喻

#### L2 状态层
- exploration_count（探索点）
- relic_count（信物/宝物）
- collectible_count（数字藏品）
- rights_count（权益）
- 当前区域：爱企谷

- 推荐探索点（3~5个滚动展示）

#### L3 行为层
- 微信一键登录（主CTA，最强视觉）
- 进入地图
- 查看信物
- 查看回响

---

## 四、PAGE_02 探索地图

### L1
- 世界地图（爱企谷及扩展区域）

### L2
- 节点系统（已解锁/未解锁）
- 推荐路径

### L3
- 点击进入探索
- AR触发
- 收藏路径

---

## 五、PAGE_03 信物系统（Relic）

### 定义：
信物 = 世界掉落的真实存在物（非数字藏品）

### L1：
- 星图 / 经络 / 象征空间背景

### L2：
- 信物列表
- 每个信物包含：
  - 名称（星名/穴位/象征）
  - 来源地点
  - 获取方式
  - 状态（未获得/已获得）

### L3：
- 查看回响入口

---

## 六、PAGE_04 回响系统（Echo）

### 定义：
信物触发后的精神反馈

### L1：
- 柔性光环境

### L2：
- 回响文本（哲理/祝福）
- 来源信物绑定

### L3：
- 收藏
- 再次探索

---

## 七、PAGE_05 数字藏品

### 定义：
探索行为的视觉纪念化记录

### L1：
- 展柜背景

### L2：
- 三类内容：
  - 探索照片
  - AR合影
  - 活动卡片

### L3：
- 生成 / 分享 / 下载

---

## 八、PAGE_06 个人中心

### L1：
- 用户身份（游客/探索者）

### L2：
- 探索次数
- 信物数量
- 回响数量

### L3：
- 我的探索
- 我的信物
- 我的收藏

---

## 九、全局数据结构（必须绑定）

user_state {
  user_type
  exploration_count
  relic_count
  collectible_count
  rights_count
  region = "aigugu"
}

---

## 十、冻结原则

- 禁止UI随意增删页面结构
- 禁止弱化状态层
- 禁止移除登录CTA
- 所有UI必须绑定数据层
