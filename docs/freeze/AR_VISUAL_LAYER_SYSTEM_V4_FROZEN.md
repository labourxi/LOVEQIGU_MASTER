# =========================
# 0. SYSTEM OVERVIEW
# =========================

AR游伴视觉系统分为四层：

L0：入口层（Landing Entry）
L1：现实增强层（Reality Bridge Layer）
L2：系统交互层（Product UI Layer）
L3：世界沉浸层（AR / Myth Layer）

---

# =========================
# 1. L0 — ENTRY LAYER
# =========================

## 1.1 SYSTEM ENTRY（微信搜索）

### 目标
- 品牌识别
- 产品介绍
- 登录引导

### 视觉
- 文旅现实风（主）
- 轻微金色/温润光效
- UI清晰、结构完整

### 禁止
- 强幻想
- 完全星空化
- 游戏化界面

---

## 1.2 SCENE ENTRY（扫码入口）

### 目标
- 立即进入场景体验
- 强地点绑定

### 视觉
- 场景真实照片 + 微增强光效
- 强地点感（如云间坊/园区入口）
- 入口按钮极简（1-2个）

---

# =========================
# 2. L1 — REALITY BRIDGE LAYER（关键规则层）
# =========================

## 2.1 定义

L1不是页面，是"视觉约束规则层"。

## 2.2 核心作用

作为 L0 → L2 / L3 的视觉缓冲带。

---

## 2.3 视觉规则（必须遵守）

### 风格比例
- 70% 现实文旅感
- 30% 微世界观增强

### 禁止行为
- 禁止完全进入幻想风
- 禁止UI游戏化
- 禁止星空/神域全覆盖
- 禁止强叙事压制UI

### 允许行为
- 微光粒子
- 弱符号化AR提示
- 光晕/能量环（低强度）
- 信物轻提示

---

## 2.4 L1本质

现实世界 + 微超现实提示系统

---

# =========================
# 3. L2 — SYSTEM UI LAYER
# =========================

## 3.1 作用

用户进入后的主操作系统

---

## 3.2 必须包含模块

- 探索首页
- 地图系统
- AR入口按钮（核心）
- 权益中心
- 个人中心

---

## 3.3 UI要求

- 清晰
- 大字号
- 适老化
- 信息分层明显
- 不拥挤

---

## 3.4 风格要求

继承L1规则：
- 现实为主
- 世界观为弱增强背景逻辑

---

# =========================
# 4. L3 — WORLD LAYER
# =========================

## 4.1 作用

AR体验层 / 信物系统 / 世界叙事

---

## 4.2 风格允许

- 星图
- 结界
- 时空门
- 经络/星宿系统
- 强神秘感
- 高幻想表达

---

## 4.3 UI规则

- UI最小化
- 体验优先
- 沉浸优先

---

# =========================
# 5. 核心流转规则
# =========================

## 用户路径

```
L0 → L1（视觉缓冲） → L2（系统） → L3（体验）
```

## 强制规则

- L0决定"期待"
- L1决定"过渡"
- L2提供"操作"
- L3提供"沉浸"

---

# =========================
# 6. 统一原则
# =========================

## 6.1 一致性原则

所有层级必须满足：

> 文旅真实世界 + 微世界观增强

---

## 6.2 世界观释放节奏

不能一次性释放：

| 层级 | 世界观释放比例 |
|------|--------------|
| L0 | 5% |
| L1 | 10% |
| L2 | 20% |
| L3 | 100% |
