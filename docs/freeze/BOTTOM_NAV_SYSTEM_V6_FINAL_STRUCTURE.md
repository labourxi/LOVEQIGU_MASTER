## 4. 硬性规则

```
┌─────────────────────────────────────────────────────────────────────┐
│ RULE-001                                                           │
│ AR is NOT a page, it is a system-level action layer                │
│ ─────────────────────────────────────────────────────────────────── │
│ AR 触发不是页面，是整个系统的行为层。                                │
│ 它作为一个覆盖层 / 行为入口存在于底部导航中，但不对应独立页面。      │
│ AR 行为可以从 Discovery、Map、或外部入口触发。                      │
│ 禁止将 AR 设计为传统意义上的 "页面"。                                │
├─────────────────────────────────────────────────────────────────────┤
│ RULE-002                                                           │
│ Discovery and Map are dual entry systems                           │
│ ─────────────────────────────────────────────────────────────────── │
│ 探索 和 地图 是双入口系统。                                         │
│ 两者都是进入世界内容的主要入口，但侧重点不同：                       │
│   - Discovery：叙事驱动、内容流、故事节点                             │
│   - Map：地理位置驱动、空间感知                                      │
│ 两者应共享数据层，但提供不同的用户体验。                             │
├─────────────────────────────────────────────────────────────────────┤
│ RULE-003                                                           │
│ Rewards is tied to AR + Discovery actions                          │
│ ─────────────────────────────────────────────────────────────────── │
│ 权益中心与 AR 行为和探索行为绑定。                                   │
│ 权益不是独立功能，而是探索/AR 行为的产出。                          │
│ 用户通过探索世界和完成 AR 互动获得权益（优惠券、奖励）。            │
│ 权益中心的视觉设计应反映其 "奖励产出" 属性。                        │
├─────────────────────────────────────────────────────────────────────┤
│ RULE-004                                                           │
│ Profile is unified asset system                                    │
│ ─────────────────────────────────────────────────────────────────── │
│ "我的" 是统一资产系统入口。                                         │
│ 信物（Relic）、数字藏品（Digital Collectible）在此统一管理。        │
│ Profile 是用户所有资产的展示和管理中心。                             │
│ 禁止将资产系统拆散到多个页面入口。                                  │
├─────────────────────────────────────────────────────────────────────┤
│ RULE-005                                                           │
│ No additional nav items allowed                                    │
│ ─────────────────────────────────────────────────────────────────── │
│ 此 5 项底部导航结构是最終版本。                                     │
│ 不允许添加额外的底部导航项。                                        │
│ 任何新功能必须容纳在此 5 项导航结构中，或作为子页面 / 模态层存在。  │
└─────────────────────────────────────────────────────────────────────┘
```
