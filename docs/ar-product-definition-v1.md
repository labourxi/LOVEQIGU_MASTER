# AR游伴 产品定义 V1

## 一、产品本质

AR游伴不是游戏，不是AR展示，而是：

> 一个基于空间的信物记忆系统

## 二、核心机制

### 1. 空间触发
用户进入特定空间后触发 XR 引擎

### 2. 仪式生成
系统通过仪式层引导用户进入探索状态

### 3. 信物生成
用户行为触发信物痕迹生成

### 4. 信物意义
信物不是奖励，是用户与空间发生关系的记录

### 5. 运营注册
所有信物进入运营层，可用于：
- 景区活动
- 内容运营
- 用户路径分析

## 三、系统边界

不包含：
- 抽卡
- 游戏化数值
- 等级体系

仅包含：
- 空间
- 事件
- 记忆
- 关系

## 四、对外解释一句话

AR游伴 = 让用户在真实世界中留下可被记住的探索痕迹

## 五、接入对象

- 景区
- 商家活动
- 城市文化点位
- 展览 / 展馆

## 六、系统输出

- 信物
- 空间锚点
- 探索记录
- 行为轨迹
