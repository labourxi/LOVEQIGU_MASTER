## §1 七个核心问题

以下回答 **只引用 Canon 已述本质** 与 **已批准的产品结构**；具体节点名、信物名、AR 码 **不在本层登记**。

---

### 1. 核心觉察

**在已初醒的探索者身上，继续看见「连接的回响」，而非获得新的力量。**

| 维度 | CH02 方向 |
|------|-----------|
| Canon 对齐 | 第四律（世界是一面回响之镜）· 第十一律（信物 = 世界留下的回响）· 第七律（看见开启道路） |
| 觉察主题 | **回响** — 探索者已在 CH01 完成第一次看见；CH02 让「连接曾被看见」继续被 **记存与辨认** |
| 产品表达 | 阈值场域中的到场 · 印谱记存 · 章成确认 · 非等级提升 |
| 「山门」边界 | L2 章节题名 · 指 **阈值/门阈** 的产品化称呼 · **不是** 新地理 · **不是** 云门本体（云门仍是状态） |

核心觉察句（方向，非最终 copy）：

> 门阈之后，回响仍在。探索不是向前夺取，而是辨认连接如何被世界记存。

**不得** 将核心觉察写成：升级、通关、收集、排名、觉醒等级。

---

### 2. 探索目标

**让探索者在真实场域中，完成本章的到场、看见与记念确认 — 与遗忘继续重逢。**

| 目标 | 说明 | 层级 |
|------|------|------|
| 到场 | 在已登记探索点完成探索记录 | L2 |
| 看见 | 延续云门状态下的觉察，不新增 Cosmology | L3 |
| 记存 | 本章 **独立印谱** 进度推进 | L2 |
| 章成 | 本章觉察结构齐备后的确认 | L2 |
| 可选结缘 | 探索完成后于结缘商城触达 · 与仪式链分离 | L1 |

探索目标 **不是**：

- 解锁新势力或新地图叙事  
- 跨章合并印谱  
- 用信物数量驱动下一章  
- 以 AR 特效本身作为目标  

`[Canon未述·暂停]`：CH02 具体探索点数量、GPS 坐标、园区命名 — 待 L2 配置，不在 Content Canon 发明。

---

### 3. 与 CH01 关系

**CH02 是 CH01 之后的并列章容器，不是世界观续集。**

| 关系 | 定义 |
|------|------|
| 时间序 | CH01《云间初醒》章成之后 · 探索继续 |
| 叙事序 | **无** Canon 级新历史事件 · 同一合真纪元 · 同一探索者 |
| 结构序 | 复制 CH01 工厂：**卷 → 章 → 节点 → 信物 → 印谱 → 章成** |
| 印谱 | **独立印谱 B** · 与 CH01 印谱 A **不合并、不跨章计数** |
| 信物 | 新章记念 · **不** 改写 CH01 信物本质 · **不** 使 CH01 信物「升级」 |
| 入口 | Story Layer `next_chapter` 由 CH01 指向 CH02 `[L2 待接线]` |
| IA | 我的信物 · 探索档案 · 数字藏馆 **追加章分组** · Tab 结构不变 |

```text
CH01 云间初醒 ──章成──▶ CH02 山门回响 ──章成──▶ [CH03+ Canon未述·暂停]
     印谱 A                    印谱 B
```

**CH02 不得** 依赖「解释 CH01 未言之秘」来成立。

### 4. 章成定义

**章成 = 本章内已定义的觉察结构全部齐备后，探索者主动确认的一次记念归位。**

| 项 | CH02 规则 |
|----|-----------|
| 触发 | 本章全部 **必需节点/修习位** 完成 · 具体清单 `[L2 待登记]` |
| 动作 | **确认章成**（L2 产品流程名）· 对齐第十二律「合真即归位」 |
| 气质 | L3 仪式 moment · 觉察语 · 记念 · **无** 积分/成就/稀有度主视觉 |
| 产物（方向） | 章级 **章成印记** · **探索记念** · **记念称号** · 本章印谱完成 |
| 产物（禁止） | 战力、境界、总印谱、跨章合成、自动写入数字藏品库 |
| 结构参考 | 与 CH01 相同工厂：**五处觉察** 或同等章内结构 `[L2 待登记]` · 不在 Content Canon 固定 n |

章成 **不是** 获得更强能力，而是 **确认** 本章探索记念已齐备。

章成后可选：分享海报（Digital Collectible 流程）· 结缘商城触达 · 返回探索地图 — 均走已有路由。

### 6. AR 事件方向

**AR 事件继续作为场域体验与 closure 入口，承接「看见门阈处的回响」，不创造云门。**

| 方向 | 规则 |
|------|------|
| Canon | 第七律 · 第四律 ·《关于云门》 |
| 职责 | 阈值场域可视化 · Story Flow closure · 预览/场域体验 |
| 交互方向 | 与 CH01 同类：`location_gate` · `awareness_prompt` · `guide_sequence` · `completion_scene` 等产品交互 **枚举复用** |
| 权限 | 默认 `camera_enabled: false` · 预览优先 |
| 绑定 | 每事件绑定 Story 节点 · 每事件至少关联 Relic **方向**（具体 ref `[L2 待登记]`） |
| 命名 | `[L2 待登记]` · 本文件 **不** 登记 `ar_*` id 或 code |
| Closure | 可接入已批准 Path A/B closure 链 · **不** 新增 Lore 型 AR 类别 |

AR 事件 **不得**：

- 宣称改运、医疗、派系、传承  
- 以 live AR 权限作为主叙事  
- 替代 L2 节点 JSON 为内容源  
- 在 n4 类修习位绑定 L1 权益弹窗

### 7. 禁止事项

#### 7.1 Canon 级（全局 · 本章同样适用）

- 新增神、文明、组织、创世神话、历史事件  
- 填补 `CANON_GAP_REGISTRY` 中 UNDECLARED / FORBIDDEN 项  
- 修改或间接覆盖 L0 冻结条文  
- 把云门写成地点、物件或可被占有的关卡  

#### 7.2 资产级

- 信物 ≠ 数字藏品 · 不得共享 progression logic  
- 仪式链不得插入结缘商城主视觉  
- Relic 不得使用 rarity / level / rank / equipment 语义  
- Rights Layer 不得伪装为章成奖励  

#### 7.3 章节级（CH02 专禁）

- **不得** 跨章合并 CH01 与 CH02 印谱  
- **不得** 重写 CH01 信物含义或 CH01 章成结果  
- **不得** 为 CH02 发明「总进度」「觉醒等级」「章节战力」  
- **不得** 在本 Content Canon 层生成 JSON、信物实体、权益实体、AR 实体  
- **不得** 用「山门」引入新地图、新国家、新文明史  
- **不得** 预写 CH03+ 叙事作为 CH02 成立条件  

#### 7.4 语言级

| 使用 | 禁止 |
|------|------|
| 探索 · 觉察 · 回响 · 信物 · 章成 · 记念 | 打卡 · 成就 · 升级 · 抽卡 |
