# =========================
# 0. 使用范围说明
# =========================

## 0.1 本文件定位

本文件**不是**设计规范，而是"规范调度规则"。

## 0.2 强制引用场景

Cursor 必须在以下情况强制引用本文件：

- 任何 UI/UX 设计任务
- 任何 Landing Page 或 Scene Page 设计
- 任何视觉风格判断或冲突解决
- 任何生图（豆包 / Gemini / Seedream）任务
- 任何页面结构设计（L0-L3）

## 0.3 不适用的场景

本文件不适用于以下情况：

- 纯代码逻辑（API / 数据 / 状态管理）
- 性能优化
- 非视觉 bug 修复

## 0.4 使用方式

Cursor 在执行任务前必须：

```
A. 判断任务是否属于视觉/结构任务
B. 若属于 → 强制加载本治理体系
C. 根据任务类型定位 L0/L1/L2/L3
D. 再引用对应具体规范执行
```

## 0.5 本文件作用

作为"规范选择器"，决定：

- 使用哪个视觉规范
- 使用哪个结构规范
- 是否允许进入世界观层（L3）

## 1. 核心规则

### 规则 A — 层级优先

所有视觉/设计相关任务必须首先识别目标所属层级：

| 层级 | 名称 | 视觉方向 | 世界观释放 |
|------|------|---------|-----------|
| **L0** | 入口层（Landing Entry） | 文旅现实风（主）+ 微金色光效 | 5% |
| **L1** | 现实增强层（Reality Bridge） | 70% 现实文旅 / 30% 微世界观增强 | 10% |
| **L2** | 系统交互层（Product UI） | 继承 L1，清晰/大字号/适老化 | 20% |
| **L3** | 世界沉浸层（AR / Myth） | 星图/结界/时空门/强幻想 | 100% |

### 规则 B — 规范引用优先级

当多个规范文件冲突时：

```
AR_VISUAL_LAYER_SYSTEM_V4 (本体系引用的根规范)
  └── 本文件 (AR游伴视觉与生产规范治理体系 V1)
        └── docs/freeze/ 下的其他冻结文件
              └── docs/architecture/
                    └── 源代码
```

### 规则 C — 输出约束

所有涉及视觉/设计的任务输出，必须：
1. 明确声明目标层级（L0/L1/L2/L3）
2. 明确声明合规状态（COMPLIANT / NON-COMPLIANT / PARTIALLY COMPLIANT）
3. 如 NON-COMPLIANT，必须指明违反的具体条款

## 2. 层级规范索引

### L0 — 入口层（Landing Entry）

**源文件**: `docs/freeze/AR_VISUAL_LAYER_SYSTEM_V4_FROZEN.md §1`

**视觉约束**:
- 文旅现实风为主
- 轻微金色/温润光效
- UI清晰、结构完整
- 禁止：强幻想 / 完全星空化 / 游戏化界面

**必含元素**:
- 品牌识别
- 产品介绍
- 登录引导（微信一键登录，主CTA）
- 底部导航（探索/地图/AR/权益/我的）

---

### L1 — 现实增强层（Reality Bridge Layer）

**源文件**: `docs/freeze/AR_VISUAL_LAYER_SYSTEM_V4_FROZEN.md §2`

**约束规则**:
- 70% 现实文旅感 / 30% 微世界观增强
- 禁止完全进入幻想风
- 禁止UI游戏化
- 禁止星空/神域全覆盖
- 禁止强叙事压制UI

**允许行为**:
- 微光粒子
- 弱符号化AR提示
- 光晕/能量环（低强度）
- 信物轻提示

---

### L2 — 系统交互层（Product UI Layer）

**源文件**: `docs/freeze/AR_VISUAL_LAYER_SYSTEM_V4_FROZEN.md §3`

**必含模块**:
- 探索首页
- 地图系统
- AR入口按钮（核心）
- 权益中心
- 个人中心

**UI要求**:
- 清晰
- 大字号
- 适老化
- 信息分层明显
- 不拥挤

---

### L3 — 世界沉浸层（AR / Myth Layer）

**源文件**: `docs/freeze/AR_VISUAL_LAYER_SYSTEM_V4_FROZEN.md §4`

**风格允许**:
- 星图 / 结界 / 时空门 / 经络/星宿系统
- 强神秘感 / 高幻想表达

**UI规则**:
- UI最小化
- 体验优先
- 沉浸优先
