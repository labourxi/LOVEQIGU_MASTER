# 1. 审查结论摘要

| 维度 | 现状 | 缺口等级 |
|------|------|----------|
| 文档与 Schema | ✅ 已有 MVP 范围、5 个 schema + mock | 字段层基本覆盖 |
| 页面骨架 | ⚠️ 7 页静态 HTML mock | **无真实交互与 API** |

## 3.1 卡券管理

| 能力 | 现状 | 优先级 | 说明 |
|------|------|--------|------|

# 4. 跨模块 P0 缺口（上线必须）

```text
1. 商家登录 + merchant_id 绑定 + 基础 RBAC（owner / staff）
2. 卡券：新建 → 提交审核 → 查看状态（含库存/有效期/权益）
3. 核销：独立页 · 扫码/输码 · 校验 · 今日/历史记录
4. 运营数据：真实 API 驱动的领取/核销/核销率（非硬编码 mock）
5. 活动申请：查看可参与活动 · 申请参与 · 选择投放卡券
6. 帮助中心：核销与常见问题（内容可静态，但必须可达）
7. 工单：可提交 + 可查看状态
8. 账单：只读列表 + 状态（待付/已付）— 首场可线下结算
9. 账号：资料查看 + 改密
10. 后端 API + 数据持久化（当前零 API）
```
