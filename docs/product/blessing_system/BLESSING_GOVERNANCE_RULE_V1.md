# Purpose

建立祝福资产治理规则。

确保未来新增祝福内容：

* 不重复
* 不鸡汤化
* 保持东方感
* 适合数字藏品展示
* 保持长期运营质量

```text
BLESSING_LIBRARY_V1

BLESSING_LIBRARY_V2+

AR显现祝福
```

---

# Governance Rule 01

重复度审查

新增祝福进入正式资产库前必须检查：

```text
是否与已有祝福表达同一含义
```

例如：

```text
慢慢变好

越来越好

终将变好
```

视为同类表达。

避免大量同义祝福进入正式库。

原则：

```text
同一情绪簇保留最佳表达。
```

---

# Governance Rule 02

鸡汤化审查

禁止出现明显互联网鸡汤风格内容。

例如：

```text
坚持就是胜利

努力一定成功

逆风翻盘

登上巅峰

你是最棒的
```

不建议进入正式资产库。

优先采用：

```text
东方表达

克制表达

安顿表达
```

例如：

```text
心安即是归处

不慌不忙

从容重启

松弛自在
```

---

# Governance Rule 03

东方感审查

新增祝福应符合：

```text
东方图谱

探索收藏

非游戏化
```

方向。

原则：

```text
像古人会说的话

而不是公众号会说的话
```

优先：

```text
静水长流

清欢自得

松风入怀

身心皆安
```

避免：

```text
未来一定更好

成为更优秀的自己

人生终将成功
```

---

# Governance Rule 04

数字藏品展示审查

新增祝福必须适合：

```text
数字藏品

分享页

祝福卡

AR显现界面
```

展示。

推荐长度：

```text
4字

6字

8字
```

优先。

避免长句。

例如：

```text
万事可期

自由如风

苦尽甘来
```

优于：

```text
希望未来的你依旧能够坚持初心
```

---

# Governance Rule 05

情绪价值审查

所有祝福必须指向正向生命状态。

允许：

```text
丰盛

安定

坚定

温暖

清明

勇气

守护

福气

自由

希望
```

禁止：

```text
焦虑

恐惧

匮乏

竞争

炫耀

优越感
```

导向。

---

# Governance Rule 06

LOVEQIGU一致性审查

所有新增祝福必须符合：

```text
LOVEQIGU_CORE_PHILOSOPHY_V1

LOVEQIGU_AXIOM_V1

BLESSING_CANON_V1
```

原则。

核心要求：

```text
祝福不是奖励。

祝福是回应。

祝福不是说教。

祝福是陪伴。

祝福不是压力。

祝福是安顿。
```

---

# Admission Rule

新增祝福进入正式资产库前必须通过：

```text
1. 重复度审查

2. 鸡汤化审查

3. 东方感审查

4. 数字藏品展示审查

5. 情绪价值审查

6. LOVEQIGU一致性审查
```

全部通过后：

```text
ALLOW_TO_ENTER_BLESSING_LIBRARY = YES
```

否则：

```text
ALLOW_TO_ENTER_BLESSING_LIBRARY = NO
```

```text
是LOVEQIGU长期内容资产。

也不允许内容质量下降。

每一条进入正式库的祝福，
```
