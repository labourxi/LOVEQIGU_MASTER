# SCENIC_OPERATION_ONBOARDING_V1

## 1. Product Decision

- scenic_operator_led_model: 平台销售 + 景区负责人 + 景区商家交流会 + 项目微信群 + 腾讯会议培训 + 培训考试 + 活动上线 + 首场活动运营
- onboarding_goal: 建立完整景区运营落地 SOP，覆盖景区、商家、店员、平台运营

## 2. Phase 1 - 景区签约SOP

### 内容

- 销售拜访
- 景区 Presentation
- 试运营确认
- 试运营协议
- 项目负责人确认
- 微信群建立

### 关键产出

- 景区合作意向确认
- 试运营时间确认
- 项目负责人和平台运营建立联系
- 项目微信群建立完成

## 3. Phase 2 - 商家招募SOP

### 内容

- 景区负责人召集商家
- 商家交流会
- 商家版 Presentation
- 商家意向登记
- 种子商家筛选

### 推荐数量

- 5 ~ 20 家

### 推荐类型

- 咖啡
- 餐饮
- 文创
- 书店
- 手作馆

## 4. Phase 3 - 项目群SOP

### 项目群名称

- XX景区项目群

### 群成员

- 平台运营
- 景区负责人
- 商家负责人

### 群职责

- 培训通知
- 活动通知
- 问题反馈
- 运营公告
- 活动复盘

## 5. Phase 4 - 腾讯会议培训SOP

### 培训对象

- 景区负责人
- 商家负责人
- 店员

### 培训时长建议

- 景区负责人：30 分钟
- 商家负责人：30 分钟
- 店员：15 分钟

### 培训目标

- 明确活动流程
- 明确授权与核销方式
- 明确异常时找谁

## 6. Phase 5 - 培训考试SOP

### 考试对象

- 景区负责人
- 商家负责人
- 店员

### 通过标准

- 80 分

### 未通过处理

- 重新培训

## 7. Phase 6 - 活动上线SOP

### 上线前检查

- 公众号授权
- 门店状态
- 核销员状态
- 活动状态
- 卡券状态

### 上线规则

- 全部通过后允许上线

## 8. Phase 7 - 首日驻场SOP

### 参与角色

- 平台运营现场支持
- 景区负责人协调
- 商家负责人值班

### 重点关注

- 领券
- 核销
- 异常反馈

## 9. Phase 8 - 首周运营SOP

### 每日动作

- 数据汇总
- 问题收集
- 商家反馈

### 输出

- 首周运营日报模板

## 10. RACI Matrix

### 角色

- 平台运营
- 景区负责人
- 商家负责人
- 店员
- 游客

### 职责矩阵

| 事项 | 平台运营 | 景区负责人 | 商家负责人 | 店员 | 游客 |
|---|---|---|---|---|---|
| 项目签约 | R | A | - | - | - |
| 商家招募 | R | A | C | - | - |
| 培训组织 | R | A | C | C | - |
| 活动发布检查 | R | A | C | - | - |
| 现场核销 | C | C | A | R | C |
| 异常反馈 | A | R | R | R | C |
| 活动复盘 | A | R | C | C | - |

> 说明：A = 负责最终结果，R = 实际执行，C = 需要协商，- = 无需参与

## 11. 风险清单

### TOP20 运营风险

1. 景区负责人未确认试运营
2. 项目微信群未建立
3. 商家参与意愿不足
4. 商家交流会未召开
5. 商家未完成扫码授权
6. 店员未完成培训
7. 店员不会核销
8. 活动时间未明确
9. 门店状态未审核通过
10. 核销员未配置
11. 卡券内容未确认
12. 卡券审核未通过
13. 活动发布前检查未完成
14. 首日驻场未安排
15. 领券链路不顺畅
16. 核销链路不顺畅
17. 游客无法找到活动入口
18. 运营公告未同步
19. 异常反馈无人接收
20. 首周日报未输出

### TOP20 应对方案

1. 建立试运营确认表
2. 建立项目群模板
3. 准备商家交流会材料
4. 准备商家意向登记表
5. 提供授权二维码
6. 提供店员核销 SOP
7. 提供核销演示
8. 活动上线前强制检查
9. 门店审核状态提示
10. 核销员状态提示
11. 平台代办卡券草稿
12. 平台代跟微信审核
13. 发布前检查清单
14. 首日驻场责任人排班
15. 领券页简化
16. 核销页简化
17. 活动入口固定展示
18. 群公告模板统一
19. 问题反馈模板统一
20. 首周日报模板统一

## 12. Design Conclusion

景区运营落地的核心不是“堆功能”，而是“把景区、商家、店员、平台运营四方的责任拆清楚，把上线前准备和上线后支持做成 SOP”。
