## 0. 设计原则

| 原则 | 说明 | 参考 |
|------|------|------|
| **东方克制** | 低饱和、纸感背景、墨色文字、点缀赭石/松绿 | 飞书 · 微信 |
| **企业级** | 表格优先、状态可扫读、操作可预期 | 企业微信 · 有赞 |
| **留白** | 模块间距 ≥16px · 页边 ≥24px · 不堆满 KPI | 飞书 |
| **角色分层** | 平台 / 园区 / 商家 三后台 **同 DS、不同导航** | 抖音来客 |
| **低技术友好** | 商家端全中文 · 禁止 token/API 外露 | 有赞 · 本项 onboarding |

**禁止：** 霓虹色 · 动效粒子 · 游戏等级/勋章 UI · 二次元插画作为主视觉

## 1. 导航体系

### 1.1 架构总览

```text
┌─────────────────────────────────────────────────────────────┐
│  Top Navigation（产品标识 · 环境 · 用户 · 全局动作）            │
├──────────┬──────────────────────────────────────────────────┤
│          │  Breadcrumb（路径 · 不可点击的当前页）                 │
│  Side    ├──────────────────────────────────────────────────┤
│  Nav     │  Page Header（标题 · 描述 · 主操作）                 │
│          ├──────────────────────────────────────────────────┤
│          │  Content（Dashboard / Form / Table / Detail）      │
└──────────┴──────────────────────────────────────────────────┘
```

三后台 **共用组件规范**，**导航项按角色裁剪**。

## 2. 页面布局规范（Desktop First）

### 2.1 栅格与断点

| 断点 | 宽度 | 内容区 max-width | 侧栏 | 列 |
|------|------|------------------|------|-----|
| **XL** | 1920 | 1600px | 240px | 12 |
| **L** | 1440 | 1280px | 240px | 12 |
| **M** | 1280 | 1120px | 240px | 12 |
| **S** | 1024 | 960px | 64px 折叠 | 12 |

**页边距：** XL/L 24px · M 20px · S 16px  
**栏间距：** 16px（卡片 gap）· 24px（区块 gap）

### 2.2 页面模板

| 模板 | 用途 | 结构 |
|------|------|------|
| **Dashboard** | 总览 | KPI 行 → 图表区 → 表格/快捷入口 |
| **List** | 列表 | Filter 条 → Table → Pagination |
| **Detail** | 详情 | 信息卡 2 列 → 关联 Tab → 操作栏 sticky bottom |
| **Form** | 创建/编辑 | 分步 Step（≥6 字段）或单页分组表单 |
| **Action** | 核销/审核 | 单焦点 + 大按钮（商家核销专用） |

### 2.3 响应式策略

```text
1920/1440  完整 Side + 4 列 KPI + 双列图表
1280       完整 Side + 3 列 KPI + 单列图表
1024       折叠 Side + 2 列 KPI · Table 横向 scroll
<1024      不在 V1 范围（商家核销另做 H5 窄屏模板）
```

**参考：** 飞书后台 1280 仍保持可读；有赞 1024 为商家最低支持宽度

## 3. 数据看板规范

### 3.1 KPI Card

| 属性 | 规范 |
|------|------|
| 尺寸 | min-height 96px · span 3/12（4 列）或 span 4/12（3 列） |
| 结构 | 标签（13px muted）→ 数值（28px semibold）→ 副文案（12px）→ 可选环比 |
| 标签 | **中文** · 如「今日核销」非 `today_redeemed_count` |
| 空态 | 「—」+ 说明「暂无数据」 |
| 点击 | 可选跳转明细 · hover 浅底 `#F5F2EC` |

**平台 KPI 示例：** 待审核数 · 今日通过 · 活跃活动 · 商家总数  
**商家 KPI 示例：** 今日领取 · 今日核销 · 核销率 · 待处理工单  
**园区 KPI 示例：** 活动参与人数 · 核销总数 · 绑定商家数 · 待发布

### 3.2 Chart

| 类型 | 场景 | 规范 |
|------|------|------|
| 折线 | 7/30 日趋势 | 单色 `#5C4033` · 网格线 `#E8E4DE` · 无 3D |
| 柱状 | 商家对比 | 最多 10 柱 · 赭石系单色梯度 |
| 饼图 | 占比 | **慎用** · 仅 ≤5 分类 |
| 占位 | 无数据 | 虚线框 + 「趋势数据准备中」 |

**高度：** 默认 200px · .dashboard 内不超过 280px  
**参考：** 微信公众平台数据页极简；抖音来客强调趋势而非炫酷

### 3.3 Table

| 属性 | 规范 |
|------|------|
| 表头 | 12px · uppercase 仅平台端内部列 · 商家端全中文 |
| 行高 | 48px |
| 斑马纹 | 不用 · 用分割线 `#E8E4DE` |
| 操作列 | 右固定 · 文字按钮「详情」「通过」「拒绝」 |
| 空态 | 插画禁用 · 文案 + 主操作引导 |
| 分页 | 底右 · 每页 10/20/50 |

### 3.4 Filter

**Filter Bar** 位于 Table 上方 · 单行优先 · 超出折叠「更多筛选」

| 控件 | 规范 |
|------|------|
| 搜索 | 圆角 8px · placeholder 中文 · 宽度 240px |
| 下拉 | 状态/类型/日期 · 标签外置 |
| 日期 | 快捷：今日/7日/30日/自定义 |
| 重置 | 文字按钮 · 右对齐 |
| 主操作 | 「导出」「新建」在 Filter 右端 |

**参考：** 有赞列表筛选项；飞书多维表格筛选克制

## 4. 表单规范

**通用规则：**
- 标签在上 · 必填 `*` · 帮助文字 12px muted
- 错误 inline 红色 · 不弹窗打断
- 主按钮右下 sticky · 次按钮「取消」
- **商家端：** 禁止 card_id / API / token 字段名

### 4.1 创建活动（园区/平台）

| 步骤 | 字段 | 控件 |
|------|------|------|
| 1 基础 | 活动名称 · 活动码 · 园区 · 时间 | Input · DateRange |
| 2 规则 | 主题 · 描述 · 参与商家上限 | Textarea · Select |
| 3 探索 | 探索点绑定 · 任务绑定 | 穿梭框 / 关联表 |
| 4 发布 | 预览 · 提交审核 | Summary Card |

**参考：** 抖音来客「创建活动」分步；字段 ≤8 个/步

### 4.2 创建卡券（商家/平台代录）

| 步骤 | 字段 | 商家端文案 |
|------|------|------------|

| 2 | 面额/折扣 · 库存 · 每人限领 | |
| 3 | 有效期 · 适用门店 · 使用说明 | |
| 4 | 预览 · 提交审核 | 「提交平台审核」 |

**平台代办模式：** 平台填 · 商家仅「确认」Checkbox

### 4.3 创建景区（平台）

| 字段 | 说明 |
|------|------|
| 景区名称 · 景区码 · 地址 · 联系人 | 基础 |
| 运营主体 · 合同档位 | 商务 |
| 状态 | 草稿 / 启用 / 停用 |

### 4.4 创建商家（平台/园区）

| 字段 | 说明 |
|------|------|
| 商家名称 · 类型 · 所属园区 | |
| 联系人 · 手机 · 营业信息 | |
| 探索点关联 · 初始卡券模板 | 可选 |
| 账号 | 手机登录 · 初始角色 owner |

**参考：** 企业微信「添加成员/部门」简化；有赞门店创建

## 5. 审核中心规范

### 5.1 状态定义

| 状态 | 英文码 | 中文 | 色 | 场景 |
|------|--------|------|-----|------|
| **Pending** | PENDING | 待审核 | `#B26B1B` 背景 `#FBEFDE` | 默认入审 |
| **Approved** | APPROVED | 已通过 | `#2C7A4B` 背景 `#E3F1E9` | 可发布/可核销 |
| **Rejected** | REJECTED | 已驳回 | `#A4412B` 背景 `#F6E0DB` | 需修改重提 |
| **Blocked** | BLOCKED | 已阻断 | `#6A5848` 背景 `#F3EADF` | 发布检查未通过 |

### 5.2 页面结构

```text
Tab：商家审核 | 卡券审核 | 活动审核
Filter：状态 · 提交时间 · 关键词
Table：名称 · 提交方 · 状态 Badge · 提交时间 · 操作
Detail Drawer：资料预览 · 审核意见 · 通过/驳回
```

### 5.3 操作规范

| 操作 | 确认 | 必填 |
|------|------|------|
| 通过 | 二次确认弹窗 | 可选备注 |
| 驳回 | 弹窗 | **驳回原因**（下拉+补充） |
| 阻断 | 发布页 | **阻断原因**清单 |

**参考：** 微信公众平台审核；飞书审批「拒绝必填理由」

## 6. 工单中心规范

### 6.1 状态定义

| 状态 | 英文码 | 中文 | 色 | 说明 |
|------|--------|------|-----|------|
| **Open** | OPEN | 待处理 | `#B26B1B` | 新提交 |

| **Closed** | CLOSED | 已关闭 | `#6F5D4D` | 归档 |

### 6.2 类型（商家/游客/园区）

| 类型码 | 中文 | 默认优先级 |
|--------|------|------------|

| COUPON_ISSUE | 卡券异常 | 高 |
| ACCOUNT | 账号问题 | 中 |
| BILLING | 账单问题 | 中 |

| OTHER | 其他 | 低 |

### 6.3 页面结构

- 列表：编号 · 类型 · 标题 · 状态 · 优先级 · 提交人 · 更新时间
- 详情：时间线 · 对话 · 附件 · 关联卡券/活动

**SLA 展示：** 待处理 >4h 标橙 · >24h 标红（仅平台/园区可见）

### 7.3 字体

| 级别 | 大小 | 字重 | 用途 |
