# 1. 文档定义

LOVEQIGU_FIRST_EVENT_MERCHANT_SCRIPT_V1，中文名为：

```text
爱企谷初见寻宝节商家招商与沟通话术 V1
```

该文档用于指导运营人员与爱企谷园区内商家沟通，完成首场活动的商家邀约、权益说明、卡券配置、核销培训、活动复盘和下一期续费沟通。

一句话定义：

```text
该文档把“爱企谷初见寻宝节”转化为商家能听懂、能参与、能核销、能复盘、能续费的沟通话术。
```

---

# 2. 上游继承原则

本文件继承以下原则：

```text
1. 商家不是广告位，而是活动节点。
2. 卡券不是商城入口，而是探索奖励。
3. 活动信物是运营资产，不能污染主线信物体系。
4. 数字藏品用于传播、纪念和凭证，不承担主线剧情。
5. 景区/园区是组织者，商家是参与者，游客是体验者。
6. 数据报告是收费和续费的关键证据。
7. 首场活动目标是跑通真实闭环，不追求复杂招商系统。
8. 商家沟通必须简单、务实、可执行。
```

本文件不得使用以下表达：

```text
上链升值
投资收藏
抢券秒杀
大促清仓
流量暴涨保证
一定带来多少成交
数字藏品升值
金融收益
```

# 3. 商家沟通目标

首场活动商家沟通的目标不是一开始高价收费，而是先让商家理解并愿意参与。

核心目标：

```text
1. 让商家知道这不是普通广告投放。
2. 让商家知道自己是园区活动的一部分。
3. 让商家愿意提供一张简单可核销卡券。
4. 让商家知道如何核销。
5. 让商家愿意看活动数据。
6. 让商家愿意参与下一期活动。
```

最重要目标：

```text
让至少 3 家商家完成参与、核销、复盘，并愿意继续参加下一期。
```

```text

获得信物后，系统会解锁参与商家的特别礼遇，比如咖啡体验券、满减券、赠品券、第二杯半价券等。

活动结束后，我们会给您一个简单数据报告，告诉您有多少人看到、多少人领取、多少人核销，方便您判断下一期是否继续参与。
```

# 9. 卡券配置沟通话术

对商家说明：

```text
建议您提供一张规则简单、店员容易理解、用户容易使用的卡券。

“满 68 减 10 元”

“消费赠小礼品”

不建议设置太复杂条件，比如多个时间段、多种商品排除、复杂会员限制。首场活动主要是先跑通领取和核销。
```

# 10. 核销流程话术

## 10.1 给商家说明

```text
用户到店后，会打开 AR游伴，出示您店铺的卡券核销码或二维码。

您只需要确认：
1. 这是本店卡券；
2. 卡券还在有效期；
3. 用户还没有核销过；
4. 完成服务或优惠后点击核销 / 扫码核销。

如果现场无法操作，可以联系活动运营人员代核销。
```

---

## 10.2 给店员说明

```text
看到用户出示“爱企谷初见寻宝节”的卡券后，请先确认是不是本店卡券。

确认后按页面提示完成核销。

如果不会操作，不要拒绝用户，先联系活动负责人处理。
```

---

## 10.3 异常处理话术

```text
如果用户说已经领取但页面打不开：
请让用户刷新小程序，或联系活动运营人员。

如果用户卡券已过期：
请按页面规则处理，不建议现场自行更改。

如果用户重复使用：
每张卡券原则上只能核销一次，请以后台状态为准。

如果商家端无法核销：
请记录用户手机号或核销码，由活动运营人员后台代核销。
```

# 11. 商家常见问题 Q&A

## Q1：这是广告吗？

```text
不是普通广告。您的店会作为园区 AR寻宝活动的一部分出现，游客完成探索后才会解锁您的礼遇。
```

## Q2：我们需要付费吗？

```text
首场活动主要是样板试点，具体参与方式可以按园区安排执行。后续如果活动效果稳定，会根据展示、卡券、核销和数据报告设计正式参与方案。
```

## Q3：会不会很多人只领券不消费？

```text
卡券必须到店核销才会计入有效数据。我们重点看的不是领取数，而是核销数和到店转化。
```

## Q4：我们要准备什么？

```text
准备一张简单卡券、一个可兑现权益、门店图片、商家名称、推荐语和一名核销负责人。
```

## Q5：用户不会操作怎么办？

```text
我们会提供活动说明、商家台卡和核销说明。如果现场遇到问题，可以联系活动运营人员处理。
```

## Q6：活动结束后能看到效果吗？

```text
可以。活动结束后会提供基础报告，包括曝光、领取、核销和用户反馈等数据。
```

## Q7：如果效果好，后面还能继续参加吗？

```text
可以。首场活动结束后，我们会根据数据和商家反馈设计下一期活动，参与商家可以优先加入。
```

# 17. 续费 / 下一期参与话术

如果商家反馈积极：

```text
从这次数据看，您的店铺已经完成了活动展示和卡券核销。

下一期我们会继续优化活动入口和探索路线，也会增加商家展示和用户引导。

如果您愿意继续参与，可以提前锁定下一期活动名额，并选择是否升级为推荐商家。
```

如果商家犹豫：

```text
首场活动主要是测试样板，数据还会继续优化。

下一期我们会根据这次问题改进活动说明、入口物料和核销流程。

您可以先继续以基础商家参与，用较低成本观察第二期效果。
```

如果商家效果较弱：

```text
这次活动中，您的卡券领取或核销数据相对不高。

我们建议下一期优化三个方面：
1. 卡券权益更直接；
2. 商家台卡位置更明显；
3. 活动页展示文案更清楚。

如果您愿意，我们可以帮您重新设计下一期卡券。
```

# 19. 商家沟通注意事项

运营人员必须注意：

```text
1. 不承诺具体成交量。
2. 不承诺一定带来多少客流。
3. 不夸大数字藏品价值。
4. 不把活动说成单纯优惠券活动。
5. 不让商家随意发行主线信物。
6. 不让商家设置无法履约的卡券。
7. 不接受过度复杂的卡券规则。
8. 不跳过核销培训。
9. 不跳过活动复盘。
```

推荐表达：

```text
我们先跑通一个小闭环，再根据数据优化下一期。
```
