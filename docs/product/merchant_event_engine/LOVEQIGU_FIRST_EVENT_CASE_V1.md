# 1. 文档定义

LOVEQIGU_FIRST_EVENT_CASE_V1，中文名为：

```text
爱企谷初见寻宝节 V1
```

它是 AR游伴在爱企谷落地的第一场真实活动样板。

该活动用于验证 MERCHANT_EVENT_ENGINE_V1 中定义的完整商业闭环：

```text
游客探索
↓
活动信物获得
↓
商家卡券领取
↓
到店核销
↓
后台数据记录
↓
活动复盘
↓
商家续费 / 景区继续组织活动
```

一句话定义：

```text
爱企谷初见寻宝节，是 AR游伴在爱企谷落地的第一个商家联动样板活动，用于验证“游客探索 → 信物获得 → 卡券领取 → 到店核销 → 数据复盘 → 商家续费”的完整商业闭环。
```

---

# 2. 上游继承原则

本活动继承 MERCHANT_EVENT_ENGINE_V1 的核心原则：

```text
1. 商家不是广告位，而是活动节点。
2. 卡券不是商城入口，而是探索奖励。
3. 活动信物是运营资产，不能污染主线信物体系。
4. 数字藏品用于传播、纪念和凭证，不承担主线剧情。
5. 景区/园区是组织者，商家是参与者，游客是体验者。
6. 活动必须模板化，后台必须按钮化。
7. 数据报告是收费和续费的关键证据。
8. MVP 先跑通真实闭环，不追求复杂营销系统。
```

本活动不得改变以下既有体系：

```text
主线信物体系
数字藏品体系
权益中心规则
探索点基础规则
世界观主线内容
```

---

# 3. 活动核心目标

本活动不追求第一期规模最大，而是追求闭环完整。

核心目标：

```text
1. 验证游客是否愿意扫码参与 AR探索活动。
2. 验证活动信物是否能提升用户完成任务的动力。
3. 验证商家卡券是否能从“探索奖励”自然转化为到店行为。
4. 验证商家是否愿意为活动曝光、卡券分发、核销数据付费。
5. 验证景区/园区管理者是否愿意支持 AR游伴成为常态活动工具。
6. 形成可复制的招商案例。
```

最重要的验收标准：

```text
不是用户领了多少卡券，
而是有没有商家愿意参加下一期。
```

# 5. 活动周期

建议第一期不要太短，也不要太长。

```text
活动周期：14 天
预热期：3–5 天
正式期：14 天
复盘期：3 天
```

执行节奏：

```text
第 -5 至 -1 天：
商家确认、物料准备、海报投放、员工培训。

第 1 至 3 天：
活动启动，重点观察扫码率和入口转化。

第 4 至 10 天：
稳定运营，观察任务完成率和卡券领取率。

第 11 至 14 天：
加强提醒，推动卡券核销和活动完成。

第 15 至 17 天：
数据复盘、商家反馈、续费沟通。
```

# 7. 活动资产设计

## 7.1 活动信物

```text
名称：企谷初见印
类型：活动信物
定位：完成基础探索任务后的核心奖励
出现条件：完成 3 个指定探索点
是否影响主线：否
是否限时：是
是否可收藏：是
```

规则说明：

```text
“企谷初见印”不是主线信物。
“企谷初见印”不参与长期主线合成。
“企谷初见印”不影响主线剧情进度。
它是爱企谷首场活动的纪念性活动信物。
```

---

## 7.2 活动数字藏品

```text
名称：爱企谷初见纪念藏品
类型：活动数字藏品
定位：完成探索 + 至少一次商家核销后的纪念奖励
出现条件：获得“企谷初见印”并完成任一商家卡券核销
用途：纪念、分享、活动完成凭证
```

注意：

```text
数字藏品不能承诺金融价值。
数字藏品不能暗示升值。
数字藏品不能替代主线信物。
数字藏品不能承担主线剧情推进功能。
```

---

## 7.3 活动卡券

第一期建议使用简单易懂的卡券。

示例：

```text
咖啡店：9.9 元咖啡体验券
茶饮店：第二杯半价券
文创店：满 68 减 10 元券
亲子体验店：体验项目 8 折券
特产店：满 50 赠小礼品券
轻餐饮店：指定套餐立减券
```

卡券规则：

```text
每人每商家限领 1 张。
活动期内有效。
必须到店核销。
核销后计入商家数据。
卡券可以进入用户权益中心。
活动页不等于权益中心。
```

---

## 7.4 活动凭证

```text
名称：初见完成凭证
类型：活动凭证
获得条件：完成 3 个探索点 + 核销任一商家卡券
用途：参与抽奖 / 活动完成证明 / 后台导出名单
```

规则：

```text
活动凭证只在本次活动周期内有效。
活动凭证可作为奖项依据。
活动凭证可由后台导出。
活动凭证可以生成核验码。
```

# 8. 探索点设计

第一期建议配置：

```text
3 个基础探索点
+
1 个商家触发点
```

## 8.1 基础探索点

```text
探索点 A：入口迎宾点
作用：活动启动、引导用户进入世界观。

探索点 B：核心景观点
作用：承载主要 AR探索体验。

探索点 C：休憩 / 转场点
作用：引导用户继续深入园区。
```

## 8.2 商家触发点

```text
探索点 D：合作商家附近
作用：完成商家到店任务，触发卡券核销和纪念藏品。
```

## 8.3 第一任务链

第一期不要做太复杂路线。

建议任务链：

```text
完成任意 3 个探索点
↓
获得“企谷初见印”
↓
解锁商家卡券
↓
核销任一商家卡券
↓
获得“爱企谷初见纪念藏品”
```

核心要求：

```text
用户必须能在较短时间内理解任务。
用户必须能在现场顺利完成任务。
商家必须能理解核销动作。
后台必须能记录关键数据。
```

# 16. 风险与控制

## 16.1 风险：用户以为只是领优惠券

控制：

```text
入口强调“寻宝”“信物”“探索”。
卡券放在任务奖励之后。
不把活动首页做成优惠券列表。
```

## 16.2 风险：任务太复杂导致用户放弃

控制：

```text
第一期只要求完成 3 个探索点。
路线短。
说明清晰。
奖励明确。
```

## 16.3 风险：商家不会核销

控制：

```text
提供核销说明卡。
活动前培训。
保留管理员代核销。
```

## 16.4 风险：数据不足，无法复盘

控制：

```text
至少记录访问、参与、完成、领取、核销五类数据。
即使人工，也必须保留表格。
```

## 16.5 风险：活动信物污染主线体系

控制：

```text
明确“企谷初见印”为活动信物。
不进入主线合成。
不影响主线剧情。
不替代主线信物。
```

```text
活动结束后，我们会给你一份简单报告，包括有多少人看到、多少人领取、多少人核销，方便你判断下一期是否继续参与。
```
