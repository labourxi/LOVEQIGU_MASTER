## 3. 分步体验模拟

### ① 扫码 — 入口

**场景：** 入口广场立牌 / 商家门口海报 / 朋友转发链接。

**小李行为：** 打开微信扫一扫。

**可能落地：**
- 静态活动页（浏览器打开）
- 小程序首页（探索 / 亲和 / 活动 Tab）
- 打不开的链接 / 404

**内心：** 「扫了，然后呢？」

| 判定 | |
|------|--|
| 会迷路 | ⚠️ **可能** — 若进小程序首页而非活动专页 |
| 会放弃 | ⚠️ **可能** — 若白屏、要下载、或跳到无关页 |
| 不知下一步 | ✅ **是** — 首页无「你正在参加寻宝节」强提示 |

---

### ② 落地 — 第一眼

**场景：** 进入活动相关页面。

**小李看到：**
- 活动名「爱企谷初见寻宝节」— 懂
- 状态「DRAFT」或英文标识 — **不懂，像没上线**
- 大段说明 + 表格 — **不想读**

**内心：** 「这活动开始了没有？是不是测试？」

| 判定 | |
|------|--|
| 会迷路 | ⚠️ 若落在小程序首页，找不到活动入口 |
| 会放弃 | ✅ **高风险** — 看到 DRAFT / 工程感文案直接关页 |
| 不知下一步 | ✅ **是** — 无「3 步玩转寻宝节」首屏引导 |

---

### ③ 理解规则 — 怎么玩

**场景：** 浏览活动页上的探索点、任务、信物列表。

**小李看到：**
- 5 个探索点名字
- 5 个任务（签到、扫码、问答…）
- 5 枚信物名字
- 一张「开始探索」按钮

**内心：** 「要先签到？还是先领券？信物是什么？能换咖啡吗？」

**缺失：**
- 没有「玩法一句话」
- 没有「推荐路线：入口 → 咖啡 → 书店 → 手作 → 中心广场」
- 没有「完成全部约需 60–90 分钟」预期

| 判定 | |
|------|--|
| 会迷路 | ✅ **是** — 信息多但无优先级 |
| 会放弃 | ⚠️ **可能** — 规则像说明书，耐心耗尽 |
| 不知下一步 | ✅ **是** — 点「开始探索」是唯一线索，但不知点后发生什么 |

---

### ④ 选起点 — 从哪开始

**场景：** 点击「开始探索」，进入探索点列表。

**小李看到：** 5 个卡片平铺，都长得差不多。

**内心：** 「先去哪？随便点一个吧。」

**行为：** 可能点「中心广场」（听起来像终点）或「爱企谷咖啡」（想喝咖啡）。

**问题：**
- 无「推荐第一站：入口广场」
- 无进度「0/5」
- 无地图「你在哪 · 点在哪」

| 判定 | |
|------|--|
| 会迷路 | ✅ **是** — 选错顺序，来回折返 |
| 会放弃 | ⚠️ 折返 2 次后开始烦躁 |
| 不知下一步 | ✅ **是** — 列表页无「下一步该点哪个」 |

---

### ⑤ 找探索点 — 现场找路

**场景：** 点了「爱企谷咖啡」，走向店里。

**小李预期：** 手机告诉怎么找、到了能做什么。

**实际：**
- 详情可能显示 **别的探索点内容**（若信息对不上）
- 只有文字坐标，无 **地图导航**
- 无「到达后请做 XXX」醒目指引

**内心：** 「这页面是不是 bug？我人都在咖啡了，它还让我看入口广场？」

| 判定 | |
|------|--|
| 会迷路 | ✅ **是** — 物理找路靠问人 |
| 会放弃 | ✅ **高风险** — 信息对不上 = 信任崩塌 |
| 不知下一步 | ✅ **是** — 到了店门口，页面上没有「我到了，签到」 |

---

### ⑥ 做任务 — 核心环节

**场景：** 站在探索点，期待扫码 / 签到 / 答题。

**小李行为：** 在页面上找按钮 — **找不到**。

**内心：** 「要扫什么码？问店员？店员也不知道这个活动。」

**现场连锁：**
- 游客问商家 → 商家未培训 → 双方尴尬
- 游客拍照发朋友圈吐槽「活动是假的」

| 判定 | |
|------|--|
| 会迷路 | ✅ **是** — 转向线下问路人/店员 |
| 会放弃 | ✅ **最高风险** — **全旅程最大流失点** |
| 不知下一步 | ✅ **是** — 页面无任务 CTA |

---

### ⑦ 得信物 — 收集感

**场景：** 假设小李没放弃，继续逛，看到信物列表。

**小李看到：** 5 枚信物名字 + 稀有度标签，**全部像「展示品」**。

**内心：** 「我拿到了吗？没提示啊。这印有什么用？」

**缺失：**
- 无「恭喜获得 XXX」 moment
- 无已收集 / 未收集对比
- 信物与领券关系不明

| 判定 | |
|------|--|
| 会迷路 | ⚠️ 不确定信物是否与自己有关 |
| 会放弃 | ⚠️ 无成就感，动力下降 |
| 不知下一步 | ✅ **是** — 不知道集信物为了领什么 |

---

### ⑧ 领卡券 — 礼遇

**场景：** 小李听说「可以领咖啡券」，找到领券页（或朋友转发链接）。

**可能路径：**
- 活动页 **无** 明显「去领券」入口 → 靠运气发现
- 领券页与活动页 **不连通**

**小李看到：** 3 张券，按钮写着「CLAIM」或「领取」。

**内心：** 「我什么都没做就能领？先领了再说。」

**领完后：**
- 「我的卡券」有记录 — 稍安心
- **没有券码、没有二维码、没有「给店员看」指引**

| 判定 | |
|------|--|
| 会迷路 | ✅ **是** — 活动页到领券页路径不清 |
| 会放弃 | ⚠️ 领完发现用不了时爆发 |
| 不知下一步 | ✅ **是** — 领完不知道怎么去店里用 |

---

### ⑨ 到店核销 — 兑现

**场景：** 小李拿手机到爱企谷咖啡柜台。

**小李：** 「我参加了寻宝活动，有咖啡券。」

**店员：** 「什么券？扫哪里？我们系统没收到。」

**小李打开页面：** 只有文字状态 CLAIMED，**无码可扫**。

**内心：** 「被骗了？」→ 差评 / 投诉 / 再也不来。

| 判定 | |
|------|--|
| 会迷路 | ✅ 游客和店员双向迷路 |
| 会放弃 | ✅ **确定放弃 + 负面传播** |
| 不知下一步 | ✅ 店员也不知道下一步 |

---

### ⑩ 活动完成 — 收官

**场景：** 即使勉强完成部分探索，也无收官反馈。

**小李内心：** 「结束了吗？还有吗？要分享吗？」

**缺失：**
- 无「恭喜完成初见寻宝」
- 无完成度 5/5
- 无纪念页 / 分享卡

| 判定 | |
|------|--|
| 会迷路 | ⚠️ 不知道是否还有隐藏任务 |
| 会放弃 | — 已在前面流失 |
| 不知下一步 | ✅ **是** — 无明确「活动已结束，感谢参与」 |

## 4. 三步判定汇总

### 4.1 在哪一步会迷路

| 步骤 | 迷路类型 | 严重度 |
|------|----------|--------|
| ① 扫码 | 进错入口（小程序首页 vs 活动页） | 🔴 |
| ③ 理解规则 | 信息过载，不知先做什么 | 🔴 |
| ④ 选起点 | 5 点平铺，无推荐顺序 | 🔴 |
| ⑤ 找探索点 | 无地图，物理找路 | 🔴 |
| ⑤ 详情 | 信息与所在位置不符 | 🔴 |
| ⑥ 做任务 | 转向问店员/路人 | 🔴 |
| ⑧ 领卡券 | 活动页找不到领券入口 | 🟠 |
| ⑨ 核销 | 游客与店员均不知操作 | 🔴 |

**最易迷路 TOP3：** ⑤ 找探索点 · ⑥ 做任务 · ⑨ 核销

---

### 4.2 在哪一步会放弃

| 步骤 | 放弃原因 | 流失风险 |
|------|----------|----------|
| ② 落地 | 看到未上线/测试感 | 🔴 高 |
| ③ 理解规则 | 规则复杂，耐心耗尽 | 🟠 中 |
| ⑤ 详情 | 信息错误，信任崩塌 | 🔴 高 |
| **⑥ 做任务** | **无操作入口，认为活动是假的** | 🔴 **最高** |
| ⑧ 领券后 | 发现无法使用 | 🔴 高（负面峰值） |
| ⑨ 核销 | 与店员冲突 | 🔴 高（投诉） |

**最大放弃点：** **⑥ 做任务**（占预估流失 **40–50%**）

---

### 4.3 在哪一步不知道下一步

| 步骤 | 「然后呢？」场景 |
|------|----------------|
| ① 扫码后 | 进了页面，第一步点哪 |
| ② 落地后 | 要不要注册/授权 |
| ③ 规则页 | 先探索还是先领券 |
| ④ 列表页 | 5 个点选哪个 |
| ⑤ 到店后 | 到了做什么 |
| ⑥ 任务页 | 怎么算完成 |
| ⑦ 信物 | 集了有什么用 |
| ⑧ 领券后 | 怎么去店里用 |
| ⑨ 核销 | 给店员看什么 |
| ⑩ 结尾 | 还有没有隐藏内容 |

**全程「不知下一步」次数：10 步中 **9 步**存在断档。**

## 9. 最小可运营体验标准（建议）

真实景区上线前，游客至少应能回答这 5 个问题：
