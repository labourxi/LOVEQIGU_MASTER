# explore_home_final_v1

## 1. 页面定位
Explore Home 不是功能首页，而是探索世界的入口首页。  
用户应该感受到：世界已经存在，但需要被慢慢发现。

## 2. 页面目标
关键词：
- 城市感
- 记忆感
- 轻叙事
- 线索感
- 轻沉浸
- 非工具化

## 3. 页面结构

### A. 顶部栏
左侧：
- 当前城市 / 状态

示例：
- 上海  今日探索

右侧：
- 搜索按钮（轻量）
- 不做复杂功能入口群

要求：
- 顶部轻
- 不工具化
- 不后台化
- 不做密集导航

### B. 探索内容主区
主区不是表格，也不是纯信息卡片列表。  
应为沉浸式探索卡片流。

每张卡片包含：

1. 雾化实景背景 / 城市场景背景
2. 探索点名称
3. 一句简短诗性描述
4. 弱提示信物数，例如：`3 个信物`

卡片示例语气：
湖面会在夜晚记住风的轨迹

### C. 卡片视觉原则
- 不强调稀有度
- 不强调等级
- 不强调奖励
- 不强调数值
- 不出现抽卡 / SSR / 成长系统感

### D. 搜索行为
点击搜索后：
- 进入轻搜索浮层
- 输入地点 / 景点 / 关键词
- 结果仍以探索卡片形式呈现

## 4. 页面氛围
Explore Home 必须让用户感受到：  
不是在看列表，而是在翻阅一张有线索的探索地图。

## 5. 动效建议
只允许：
- 卡片缓慢浮现
- 滚动轻视差
- 背景轻雾
- 状态渐显

禁止：
- 奖励飞入
- 爆闪标签
- 强交互动效
- 商城式切换
