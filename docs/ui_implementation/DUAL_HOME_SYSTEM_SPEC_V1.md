# DUAL_HOME_SYSTEM_SPEC_V1（双首页系统宪法）

## 一、系统定位（不可变）

本系统定义 LOVEQIGU / AR游伴 的"双首页结构"，用于统一所有视觉、交互与信息流的首页行为规范。

---

## 二、双首页结构定义（核心冻结）

### 1. Landing Page（入口层）

定位：世界入口（Entry Layer）  
性质：情绪驱动 / 仪式化入口页  

职责边界：
- 只负责"进入世界"
- 不承载信息分发
- 不承载内容浏览
- 不承载搜索体系
- 不承载列表结构

核心行为：
- 唯一 CTA：进入探索
- 世界氛围建立（雾 / 星 / 留白）
- 第一印象控制

禁止行为：
✘ 卡片列表
✘ 探索内容展示
✘ 搜索系统
✘ 信息流
✘ 商业结构

---

### 2. Explore Home（探索层）

定位：世界展开层（World Layer）  
性质：内容承载 / 探索入口  

职责边界：
- 城市 / 探索点展示
- 信物线索承载
- 探索行为入口
- 搜索能力入口（轻量）

核心行为：
- 卡片式探索流
- 城市级探索分发
- 轻搜索浮层

禁止行为：
✘ 单一入口仪式
✘ Landing式极简封闭结构
✘ 强品牌入口页结构

---

## 三、双首页关系（关键逻辑）

Landing → Explore（单向进入）

说明：
- Landing 是"门"
- Explore 是"世界"
- Landing 不回收信息
- Explore 不承担入口仪式

---

## 四、信息流规则

### 1. 信息分层原则
Landing = 无信息层（Pure Entry）
Explore = 有信息层（World Content）

### 2. 数据承载规则
Landing ❌ 禁止内容数据
Explore ✔ 允许全部探索数据

### 3. 搜索规则
搜索只能存在于 Explore Home

Landing 禁止搜索

---

## 五、视觉语言分层

Landing：
仪式 / 雾 / 星 / 未完成 / 留白

Explore：
城市 / 线索 / 卡片 / 记忆 / 轻叙事

---

## 六、交互层级规则

Landing：
唯一行为 = 进入世界

Explore：
探索 / 浏览 / 搜索 / 点击卡片

---

## 七、系统禁区（双层共用）

✘ 游戏化奖励
✘ SSR / 稀有度系统
✘ 抽卡结构
✘ 电商列表结构
✘ 工具后台感 UI
✘ 数值成长体系

---

## 八、可扩展原则

所有新能力必须归属 Explore
不得回流 Landing

---
