# VISUAL_PROMPT_PACK_LANDING_AND_EXPLORE_V1

项目：
LOVEQIGU / AR游伴

阶段：
VISUAL_LANDING_PRODUCTION_V1

用途：
交由半自动生图生产线执行，用于首页终版视觉稿第一批生产

总原则：
- 不新增世界观内容
- 不新增系统
- 不新增页面
- 只围绕双首页：
  1. Landing Page
  2. Explore Home
- 视觉目标：回到最初满意方向，而不是继续偏后台/工具化
- 参考基准：第一版首页气质 —— “留在足迹里，收藏世界”
- 输出必须能直接作为 Cursor 前端还原视觉依据
- 强调东方、安静、克制、梦幻、可探索
- 禁止做成电商页、管理页、强营销页、重游戏化页
- 禁止出现积分、奖励、稀有、SSR、数值成长、等级感
- 禁止五彩高饱和科技大屏感
- 禁止强按钮堆叠、复杂入口堆叠、多导航分流

统一风格关键词：
东方 / 留白 / 卷轴 / 星象 / 微雾 / 夜色 / 金石 / 古籍 / 轻梦境 / 微光 / 收藏感 / 足迹 / 世界

统一画面要求：
- 高级、安静、克制
- 低噪声
- 轻微呼吸感
- 暗色主背景 + 柔和暖金/米金点缀
- 字体排版简洁，中文为主
- 不做重运营banner感
- 不做杂乱图标宫格
- 不做复杂卡片墙
- 必须具备“可以直接还原成前端页面”的合理结构

统一视觉 token 倾向：
- 深墨黑 / 夜蓝黑背景
- 米白 / 暖灰正文
- 暖金 / 旧金作强调
- 轻雾、微粒、微光、星点
- 圆角柔和，阴影克制
- 留白充足
- 气质重于信息量

--------------------------------
A. 任务一：Landing Page
输出文件名：
landing_page_final_v1
--------------------------------

页面定位：
这是进入 AR游伴的第一张首页，不是功能入口页，不是运营活动页，不是后台页。
它承担“世界观入口 + 情绪建立 + 进入探索”的作用。

必须包含：
1. 页面中心品牌标题：
   AR游伴
2. 辅助文案：
   留在足迹里，收藏世界
3. 唯一主按钮：
   进入探索
4. 背景为深色、梦幻、东方感场景
5. 隐约出现星图/地图/探索轨迹/世界轮廓感，但不要做成真实地图UI
6. 整体是“进入探索前的一次安静召唤”

页面结构建议：
- 上方无复杂导航
- 中心为品牌与主文案
- 中下部仅一个主 CTA：进入探索
- 背景可带隐约星辰轨迹、地图轮廓、雾气、粒子、远山/卷轴式纹理
- 底部或主视觉附近弱化表达“足迹”“收藏世界”的意向

视觉气质要求：
- 安静、沉浸、梦幻、克制
- 像一个探索世界的入口
- 有“收藏世界”的诗意
- 不要产品说明堆砌
- 不要多入口
- 不要后台感
- 不要冷硬 SaaS 感

文案只允许非常少量：
- AR游伴
- 留在足迹里，收藏世界
- 进入探索

可选极少辅助字句（如需要）：
- 今日，去哪里探索？
但只可极弱使用，不可喧宾夺主

构图重点：
- 品牌居中或略偏上居中
- CTA 明确但不刺眼
- 背景层次丰富但不过满
- 整体必须一眼高级

请生成：
2个方向的小变体
方向1：更诗意、更留白、更东方
方向2：更梦幻、更沉浸、更有微光星图感

--------------------------------
B. 任务二：Explore Home
输出文件名：
explore_home_final_v1
--------------------------------

页面定位：
这是用户进入后真正开始探索的首页。
不是后台，不是数据页，不是奖励页，而是“今天去哪里探索”的首页。

必须包含：
1. 页面标题/主识别：
   上海 · 今日探索
2. 轻量搜索入口
3. 6张探索卡片（可视为推荐探索地点）
4. 每张卡片体现：
   - 地点名称
   - 一句很短的气质描述
   - 弱信物提示 / 氛围提示
   - 沉浸式背景图感
5. 页面整体突出：
   今天去哪里探索？
6. 允许有轻量定位/城市感，但不要做地图工具页

探索卡片内容倾向（可自由视觉化，不必逐字全上）：
- 外滩夜行
- 武康路午后
- 朵云书院
- 苏州河微风
- 豫园灯影
- 徐汇书角

卡片文案风格：
- 非营销
- 非攻略
- 非奖励
- 非运营任务
- 只做轻探索引导

可参考的短描述气质：
- 在夜色与风中，收下一段城市回响
- 午后的树影下，藏着一枚安静信物
- 翻开书页时，回应会悄悄亮起
- 沿河而行，微光会在足迹后出现

页面结构建议：
- 顶部：上海 · 今日探索
- 顶部右侧或相邻区域：搜索按钮/搜索框
- 主体区域：6张沉浸式探索卡片，2列或单列节奏化排布
- 每张卡片以大图氛围为主，信息很轻
- 页面整体有“城市探索杂志首页”质感，而不是工具列表页

必须避免：
- 表格感
- 后台感
- 数据 dashboard 感
- 奖励/积分/成就/任务中心感
- 过多运营角标
- 花哨营销标签
- 重功能按钮堆叠

视觉气质要求：
- 仍延续 Landing Page 的世界观气质
- 但更生活化、更可进入、更贴近真实探索
- 一眼看到“城市探索推荐”
- 一眼想点进去看详情

请生成：
2个方向的小变体
方向1：更偏东方梦境城市感
方向2：更偏杂志感、探索策展感

--------------------------------
C. 统一负面约束
--------------------------------

禁止出现：
- 积分商城
- 奖励中心
- 稀有度标签
- SSR / UR / 爆闪
- 数据报表
- 多后台入口
- 红橙黄高饱和运营促销视觉
- 杂乱 icon 栈
- 电商活动 banner
- 游戏任务中心
- 抽卡感
- 复杂 KPI 组件
- 明显代码原型感
- 低级拟物
- 通用 AI 科技蓝大屏风格

--------------------------------
D. 交付要求
--------------------------------

输出应满足：
- 可以直接给前端做高保真还原
- 页面结构清晰
- 字体、按钮、留白、层级明确
- 图片气质统一
- Landing 与 Explore Home 风格一体化
- 允许轻微差异，但应属于同一视觉体系

验收目标：
- 回到“第一版首页满意方向”
- 比当前 Demo 首页明显更高级
- 具备正式评审价值
