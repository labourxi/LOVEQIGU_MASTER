# LOVEQIGU AGENT RULES

## Mission

Build LOVEQIGU according to the official Canon, World Bible, Language Constitution, Terminology, and Information Architecture.

Agents must treat documentation as the source of truth.

---

## Knowledge Priority

Priority order:

1. docs/canon/*
2. docs/world/*
3. docs/language/*
4. docs/architecture/*
5. Source code

Higher priority documents always override lower priority documents.

---

## Canon Rules

Never invent:

* New gods
* New civilizations
* New organizations
* New creation myths
* New historical events

Never fill Canon gaps.

If information is missing, report the gap instead of creating new lore.

---

## Terminology Rules

Always use official terminology.

Required replacements:

* 打卡地图 → 探索地图
* 积分商城 → 权益中心
* 愿力 → 心愿值
* 归真 → 合真
* 回应 → 回响
* 祝由 → 祝禁

Never introduce unofficial terminology.

---

## Asset Rules

Relic (信物):

* Story progression asset
* Canon-driven asset

Digital Collectible:

* Marketing asset
* Communication asset

Relic != Digital Collectible

Never mix them.

---

## Language Constitution

Respect language layers:

L1 = Commercial Language

L2 = Product Language

L3 = Canon / Ritual / Worldview Language

Do not mix layers.

---

## Development Rules

Before any task:

1. Read all files under docs/
2. Verify terminology
3. Verify Canon constraints
4. Verify Language Constitution constraints

If a task conflicts with documentation:

STOP

Explain the conflict.

Do not implement.

---

## Output Rules

When uncertain:

* Ask
* Report
* Do not invent

Canon consistency is more important than implementation speed.
